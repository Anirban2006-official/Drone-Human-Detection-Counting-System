# 🚁 Drone Vision Pipeline
### Human & Car Detection · Counting · Tracking — VisDrone Dataset

---

## Overview

End-to-end computer vision pipeline for analysing drone/aerial imagery built with **YOLOv8** and a custom **SORT/ByteTrack** multi-object tracker.

| Task | Description |
|------|-------------|
| **01** | Dataset Understanding & Preprocessing |
| **02** | YOLOv8 Model Training / Fine-tuning |
| **03** | Human & Car Detection + Counting |
| **04** | Object Tracking (ByteTrack / SORT) *(bonus)* |
| **05** | Evaluation & Visualization |

## Dataset

**VisDrone** — large-scale benchmark for drone imagery object detection.

- 📎 [Kaggle download](https://www.kaggle.com/datasets/banuprasadb/visdrone-dataset/data)
- ~10,000 images across train / val / test splits
- 10 original classes → mapped to **2 target classes**: `person`, `car`

### Class Mapping

| VisDrone class | → | Target |
|---|---|---|
| pedestrian, people | → | **person** |
| car, van, truck, bus | → | **car** |
| bicycle, motor, tricycle, others | → | *(ignored)* |

### Key Challenges
- 🔍 **Small objects** — hundreds of < 20 px detections per image
- 👥 **Heavy occlusion** — crowded pedestrian scenes
- 📐 **Scale variance** — objects range from 5 px to 300+ px
- ⚖️ **Class imbalance** — person >> car >> others
- 🌫️ **Lighting & weather** — dawn, dusk, fog, overcast

## Usage

### Quick Start (all tasks)

```bash
python main.py \
  --dataset /path/to/VisDrone \
  --mode    all \
  --epochs  100 \
  --batch   16
```

---

### Task 01 – Dataset Analysis

```bash
python main.py --dataset /path/to/VisDrone --mode dataset
# OR
python src/dataset.py --dataset /path/to/VisDrone --split train --convert
```

**Outputs:**
- `outputs/visualizations/dataset_stats_train.png`
- `outputs/visualizations/sample_1.png` … `sample_N.png`
- `data/yolo_format/` — YOLO-formatted labels

---

### Task 02 – Model Training

```bash
python main.py \
  --dataset /path/to/VisDrone \
  --mode    train \
  --model   yolov8m.pt \
  --epochs  100 \
  --batch   16 \
  --imgsz   640 \
  --device  0          # GPU id ('' for auto)
```

**Outputs:**
- `outputs/runs/visdrone_yolov8m/weights/best.pt`
- Training curves, confusion matrix, PR curves (auto-generated by YOLO)

**Recommended models by hardware:**

| GPU VRAM | Model |
|---|---|
| ≥ 16 GB | yolov8l.pt / yolov8x.pt |
| 8–16 GB | yolov8m.pt *(default)* |
| 4–8 GB  | yolov8s.pt |
| CPU only | yolov8n.pt |

---

### Task 03 – Detection & Counting

```bash
# Single image
python src/detect.py \
  --weights outputs/runs/visdrone_yolov8m/weights/best.pt \
  --source  /path/to/image.jpg \
  --output  outputs/detections

# Image folder
python src/detect.py \
  --weights outputs/runs/visdrone_yolov8m/weights/best.pt \
  --source  /path/to/images/ \
  --output  outputs/detections

# Video
python src/detect.py \
  --weights outputs/runs/visdrone_yolov8m/weights/best.pt \
  --source  /path/to/video.mp4 \
  --output  outputs/detections
```

**Outputs:**
- Annotated images/video with bounding boxes
- Per-image CSV with class, bbox, confidence
- Console summary: total persons & cars

---

### Task 04 – Object Tracking (Bonus)

```bash
python src/tracking.py \
  --weights outputs/runs/visdrone_yolov8m/weights/best.pt \
  --source  /path/to/video.mp4 \
  --output  outputs/tracking \
  --max_age 30 \
  --min_hits 3
```

**Outputs:**
- `outputs/tracking/*_tracked.mp4` — video with persistent track IDs
- Console: unique person/car counts, average track lifetime

---

### Task 05 – Evaluation

```bash
python src/evaluate.py \
  --weights  outputs/runs/visdrone_yolov8m/weights/best.pt \
  --source   /path/to/VisDrone/images/val \
  --ann_dir  /path/to/VisDrone/annotations/val \
  --output   outputs
```

**Outputs:**
- `outputs/visualizations/pr_curves.png`
- `outputs/visualizations/count_timeline.png`
- `outputs/visualizations/evaluation_dashboard.png`
- `outputs/evaluation_report.json`

## Augmentation Strategy

Fine-tuned for drone/aerial imagery:

| Augmentation | Value | Reason |
|---|---|---|
| Mosaic | 1.0 | Exposes model to many small objects per batch |
| Copy-paste | 0.1 | Synthesises dense crowd scenarios |
| Scale | 0.5 | Handles altitude variation |
| Flip (H/V) | 0.5 / 0.3 | Aerial symmetry |
| HSV jitter | std values | Lighting/weather robustness |
| MixUp | 0.15 | Regularisation |
| Rotation | ±5° | Small tilt compensation |

## Architecture

```
Input Image (drone frame)
        │
        ▼
 ┌──────────────┐
 │  YOLOv8-M    │  ← fine-tuned on VisDrone
 │  Backbone:   │    (CSPDarknet + C2f)
 │  Neck: PAN   │
 │  Head: Decoupled │
 └──────┬───────┘
        │  raw detections [N × (bbox, conf, class)]
        ▼
 ┌──────────────┐
 │ NMS + filter │  conf > 0.30, IoU < 0.45
 └──────┬───────┘
        │
   ┌────┴────┐
   │         │
   ▼         ▼
Person      Car
Counter   Counter
   │         │
   └────┬────┘
        │  (optional)
        ▼
 ┌──────────────┐
 │  SORT Tracker│  Kalman predict → IoU match → update
 │  (per class) │
 └──────┬───────┘
        │
        ▼
 Visualizer → annotated frame + count overlay
```

## Strengths & Limitations

### ✅ Strengths
- Real-time capable on GPU (40–80 FPS)
- YOLOv8 anchor-free head handles small object diversity well
- Per-class Kalman tracking maintains identity through occlusion
- Mosaic augmentation dramatically improves small-object recall
- Clean modular codebase — easy to swap model or tracker

### ⚠️ Limitations
- Objects < 10 px remain challenging (fundamental resolution limit)
- Counting is frame-level; global re-ID across large scenes is not implemented
- Heavy crowd occlusion still causes missed detections
- CPU inference too slow for real-time drone deployment

### 🔧 Potential Improvements
- **Tiled inference** (SAHI) for very high-res images
- **BoT-SORT / StrongSORT** for better re-identification
- **Transformer-based detector** (RT-DETR) for global attention
- **TensorRT export** for edge deployment on drone hardware
